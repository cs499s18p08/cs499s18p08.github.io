Design


Overview

  • Save New Job data created by user to server
  • Search vehicle database (already owned by company)
  • Search reference material database (data has been acquired by company)
  • Navigate to create a new job, access reference materials, and access the vehicle database through the sidemenu
  • Allow for the collection of GPS coordinates for geo-referencing

Environment

The application will be designed using React-Native, an open source framework backed by Facebook. We are using Marshmallow, an Android Emulator, along with google APIs. The emulator is built and processed byan IDE called Gradle. This allows us to simulate a virtual mobile device and test how the mobile application will workas we build it. All of this will be done using Windows 10.

Languages

The languages required for the project so far:
  • Javascript
  • JSX
  • CSS

Design Flow

Start up: This page is the default page when the app starts up. The user can choose to log in if they have a signal and a login for the app or they can choose to go to the reference pages online or offline.

Log in: This page is the log in screen if the user chooses to log in at the start up page. They enter their username and password into the spots provided. They can only log in if they have set up their account through the desktop application. There is also the function for the user to reset their password through a seperate screen.

Password reset: Field asking to enter an email. When the send email button is pressed it sends an email for a confirmation code for resetting your password. Also when the send email button is pressed it sends you to a screen where you enter the confirmation code which upon being entered will bring them to a page to enter a new password.

Select a job: This is where the user can choose to selet an old job or choose to add a new job. It'll have a list of jobs with the date that they were created.

Side navigation: This window will be prompted when the user swipes to the right if they swipe from the left side of the screen. It will have options for easier navigation including: job list, reference material, uplaod data to server, settings, and a sign out option.

Settings: This screen will have general settings such as auto upload, auto sign out if they are away from the app or away from their phone. Also they will have the option to choose a theme: light or dark. They will also be able to the clear the data of the app that is saved on their phone with a button press.

Reference material: This screen will allow the users to look through databases that are organized into a nice format. They can look at the reference material page even if they have the log in or not. The type of things that they'll be able to look at are: Coefficient of friction of various roadways, default car data, low speed coefficient of restitution, pedestrian walking speed/velocities, and a vehicle database with a look up option.

General Information: This is wehre users can input basic information for the job that they are reporting. It has features such as: the job name, location with the possible use of using google maps to get the location, and what the weather was like on the day

Vehicle Information: This page includes the option to pick the number of vehicles involved in the crash. With the number chosen it will bring up multiple different prompts to fill in for each vechile. For each vehicle they can fill out the: vehicle year, make, model, and color, license plate and the inforamtion involved with it, the vin number, and other various vehcile specs involved.

Occupation Information: This page includes the option to pick the number of people involved with the screen and with the number of people chosen it will prompt the app to bring up multiple different prompts to fill in for each person. For each person they can fill out their: Name, address, phone, birthdate, vehicle they were in as well as position in vehicle, their impairment, and any other notes necessary.

Photos: With this page the user will be able to upload and take pictures of the job that they are working. They will be able to upload the photo to the database when they upload the job as well.

Personal Notes: This page will allow the users to record personal notes on the job they are working.
