Project Requirements and Specifications

  1. Work with other teams on the database and desktop application to develop the best tools for Delta V Innovations with the goal in mind for future implementation. Our goal will be to have a complete project encompassing these requirements for presentation in Marksbury Theater by the end of the Spring 2018 Semester.
  2. Work with the desktop team to define functions for operation between the mobile app and the desktop application with the goal of making it avaliable for future implementations:
    1. Save New Job data created by user to server
    2. Search vehicle database (already owned by company)
    3. Search reference material database (data has been acquired by company)
  3. The app produced must be cross platform and able to work on a variety of devices including iPhones, iPads, Android phones, and Android Tablets.
  4. Develop a tab/menu function that will include the ability to create a new job, access reference materials, and access the vehicle database.
    1. Allow for users to access the reference materials without being logged in for ease of access
  5. In the new job options have a tab for adding new vehicle, crash location, scene data, photos, and notes.
  6. Allow for the collection of GPS coordinates for geo-referencing.
  7. Back up data to the server on a set interval to ensure that signifigant amounts of data are not lost in the event of an app crash or lost phone. Allow for the user to manually back up data within the app as well and change the setting for how often they want backups to occur.
  8. Automatically fill out fields on the app by calculating common formulas based on user information inputs.
  9. Use APIs for transmitting weather information and vehicle information to the database
  10. Stretch Goal: Make a QR reader using external resources to read information off of the barcode/QR code on inside of vehicle's door and automatically fill into the form. Then send the information to the database.

Platforms

The Mobile Application will be designed to work on both Android and iOS devices.

Tools

The application will be designed using React-Native, an open source framework backed by Facebook.We are using Marshmallow, an Android Emulator, along with google APIs.
The emulator is built and processed byan IDE called Gradle. This allows us to simulate a virtual mobile device and test how the mobile application will workas we build it. All of this will be done using Windows 10.
